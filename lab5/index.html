mediaLab5 / WebRTC

Browser-to-browser video streaming

Server page captures webcam video. Client page receives and displays the stream through RTCPeerConnection.

Checking context... Manual SDP signaling

Test steps

  1. Open Server page in one browser and press Start camera and create offer.
  2. Copy the generated offer SDP from the server page.
  3. Open Client page in another browser, paste the offer, and press Create answer.
  4. Copy the generated answer SDP from the client page back to the server page.
  5. Press Connect answer on the server page.
Server

Capture video

Creates the offer and sends webcam tracks to the peer connection.

 Client

Display stream

Creates the answer and renders the received video stream.
